Open Smart Environment Bundle

Documentation for "ose-bundle" package (GitHub)

This package wraps all official OSE packages, contains examples and allows documentation building using YUIDoc.

The main advantage of OSE is the easy creation of applications consisting of multiple instances working as a single whole. The objective is to develop an all-encompassing personal mesh running various devices including HTPCs, phones, tablets, workstations, servers, Raspberry PIs, home automation gadgets, drones etc.

Status

  • Pre-alpha stage (insecure and buggy)
  • Unstable API
  • Gaps in the documentation
  • No test suite

This is not yet a piece of download-and-use software. Its important to understand the basic principles covered by the documentation.

Use of this software is currently recommended only for users that wish participate in the development process, see Contributions.

Getting started

To get started with OSE, refer to the Media player example application. You can read the entire OSE documentation here.

Installation

Node.js

The main prerequisite is a working installation of a recent installation of Node.js (>= 0.10).

On Debian Jessie, you can install the default package:

sudo apt-get install nodejs

On Raspbian, install Node.js by doing:

wget http://node-arm.herokuapp.com/node_latest_armhf.deb
sudo dpkg -i node_latest_armhf.deb

OSE Bundle Prerequisites

You also need the following prerequisites

  • The libdbus-1-dev package or its equivalent for your distribution
  • The pkg-config package or its equivalent in your distribution

If you run Debian Jessie, just run:

sudo apt-get install libdbus-1-dev pkg-config

OSE Bundle Installation

The easiest way of getting a working OSE installation is to install the ose-bundle package using npm:

npm install ose-bundle

The "ose-bundle" package contains all official OSE packages as its dependencies.

Manual installation of OSE packages

Instead of using npm, you can install OSE packages by cloning their GitHub repositories. You can thus install only the packages that you actually need.

Components

Open Smart Environment Bundle consists of the following components:

  • LIRC example application
  • Media player example application
  • Raspberry Pi example application

LIRC example application

Extends the media player example application. Allows to control the media player with a IR remote controller via LIRC.

In this example, the remote controller behaviour is configured as follows:

  • Volume up, down and mute
  • Play, pause, stop
  • Activate the audio command group by pressing "KEY_AUDIO" and select a predefined stream by pressing some digit
  • Activate the DVB by pressing "KEY_TV" command group and select a predefined DVB channel by pressing some digit
  • Switch to previous or next channel by pressing the "KEY_PREVIOUS" and "KEY_NEXT" keys.

You can configure the behaviour to suit your needs.

Read more about LIRC example application ...

Media player example application

This example is an stand-alone application based on the OSE framework, showcasing some of its principles and capabilities. This application works as a currently half-featured media player and remote control on Linux boxes. The example application has the following features:

  • Node.js backend
  • Control via HTML5 frontend instances
  • Near-realtime synchronization among all front- and backend instances
  • Keyboard and pointer remote control using xdotool
  • Playback of different media using VLC
  • Predefined media streams
  • Icecast directory search and playback
  • Youtube search and playback
  • Local files playback
  • Playback from history

Other example applications consisting of multiple instances working as a single whole are coming.

Raspberry Pi example application

This application allows to control features of the Raspberry Pi. It can be used on its own or together with other example OSE applications.
